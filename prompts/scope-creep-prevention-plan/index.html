Scope Creep Prevention Plan

Describe your project scope and client situation. Get a change request system, boundaries script, and scope protection framework.

How to Use

Fill in the project details. Use the generated framework to protect your time and set clear boundaries with clients.

The Prompt

Help me prevent and manage scope creep for my freelance project.

Project type: [WEB DESIGN / COPYWRITING / DEVELOPMENT / MARKETING / ETC.]
Original scope: [WHAT WAS AGREED UPON]
Client type: [STARTUP / CORPORATE / SMALL BUSINESS / INDIVIDUAL]
Current situation: [DESCRIBE ANY SCOPE CREEP ALREADY HAPPENING, OR "STARTING FRESH"]

Create:

1. Scope definition document template:
   - Clear list of what IS included (deliverables, revisions, timeline)
   - Explicit list of what is NOT included
   - Definition of "done" for each deliverable
   - Number of revision rounds and what counts as a revision vs. a new request

2. Change request process:
   - A polite but firm email template for when a client asks for something outside scope
   - How to frame additional work as an opportunity (not a conflict)
   - A simple change order template with: description, estimated time, cost, impact on timeline

3. Prevention strategies:
   - 5 questions to ask during the initial briefing that prevent scope creep later
   - Red flags that a client will be a scope creeper
   - How to build a buffer into your timeline and pricing without overcharging

4. Scripts for common scenarios:
   - "Can you just add one more thing?" response
   - "I thought this was included" response
   - "My colleague has some changes" response

Tone: Professional and relationship-preserving. The goal is to protect my boundaries while keeping the client happy.
