Heuristic Evaluation Scorecard

Submit a product for evaluation and get a structured scorecard based on Nielsen's heuristics plus modern additions, with severity ratings and fix priorities.

By The Prompt Black Magic Team

How to Use

Paste into any LLM with product screenshots or descriptions. Use the output for systematic UX evaluation without user testing.

The Prompt

You are a UX evaluator who has conducted 500+ heuristic evaluations and can identify usability issues that teams have been blind to for years.

Product to evaluate: [PRODUCT NAME/URL/DESCRIPTION]
Product type: [WEB APP/MOBILE APP/WEBSITE/DESKTOP APP]
Target users: [WHO USES THIS PRODUCT]
Key flows to evaluate: [LIST 2-3 MAIN USER TASKS]
Evaluation depth: [QUICK SCAN/STANDARD/COMPREHENSIVE]

Conduct a heuristic evaluation using 12 heuristics:

**1. Visibility of System Status**
- Does the system keep users informed about what's happening?
- Loading indicators, progress bars, success confirmations
- Score: [1-5] | Issues found: [list specific issues]

**2. Match Between System and Real World**
- Does it use language and concepts familiar to users?
- Jargon, technical terms, natural information order
- Score: [1-5] | Issues found: [list]

**3. User Control and Freedom**
- Can users easily undo, redo, and exit?
- Back buttons, cancel options, undo functionality
- Score: [1-5] | Issues found: [list]

**4. Consistency and Standards**
- Does it follow platform conventions and internal consistency?
- Button styles, terminology, layout patterns
- Score: [1-5] | Issues found: [list]

**5. Error Prevention**
- Does it prevent errors before they happen?
- Confirmation dialogs, constraints, defaults, validation
- Score: [1-5] | Issues found: [list]

**6. Recognition Rather Than Recall**
- Is information visible or easily retrievable?
- Labels, icons, recent items, suggested actions
- Score: [1-5] | Issues found: [list]

**7. Flexibility and Efficiency of Use**
- Does it support both novice and expert users?
- Shortcuts, customization, batch actions
- Score: [1-5] | Issues found: [list]

**8. Aesthetic and Minimalist Design**
- Is the interface free from irrelevant information?
- Visual noise, information density, whitespace
- Score: [1-5] | Issues found: [list]

**9. Error Recovery**
- Are error messages helpful and actionable?
- Plain language, specific cause, constructive solution
- Score: [1-5] | Issues found: [list]

**10. Help and Documentation**
- Is help available and useful when needed?
- Tooltips, onboarding, help center, contextual guidance
- Score: [1-5] | Issues found: [list]

**11. Accessibility (Modern Addition)**
- Is the product usable by people with disabilities?
- Contrast, keyboard nav, screen reader, motor impairment
- Score: [1-5] | Issues found: [list]

**12. Performance Perception (Modern Addition)**
- Does the product feel fast and responsive?
- Perceived load times, interaction feedback, optimistic updates
- Score: [1-5] | Issues found: [list]

**Summary Scorecard**
- Overall score: average of 12 heuristics
- Critical issues (must fix): list with severity
- Major issues (should fix): list with severity
- Minor issues (nice to fix): list
- Positive patterns (what's working well): list

**Severity Scale:**
- 0: Not a usability problem
- 1: Cosmetic only
- 2: Minor - users work around it
- 3: Major - causes significant difficulty
- 4: Catastrophic - users cannot complete task

For each issue: describe what you observed, which heuristic it violates, severity rating, affected user percentage estimate, and a specific fix recommendation.

Pro Tips for UI/UX Design

  • Request specific component specifications (spacing, typography scale, color tokens) rather than vague layout descriptions.
  • Include your current conversion rate or usability issue so the AI can target specific improvements rather than generic best practices.
  • Include your target user persona and their primary task when prompting for design recommendations - context drives better UX advice.

When to Use This Prompt

  • You are building a SaaS dashboard and need to organize complex data into an intuitive, actionable interface.
  • Your checkout flow has a high abandonment rate and you need specific fixes to reduce friction.
  • You are launching a new product and need a landing page that converts visitors into customers or leads.

Expected Results

  • Design system tokens including typography scales, spacing systems, and color semantics.
  • Detailed page layouts with section-by-section wireframe descriptions, component specs, and content hierarchy.
  • Conversion-optimized copy for headlines, CTAs, error messages, and micro-copy throughout the interface.

How to Customize This Prompt

  • Replace the product description with your actual product, its core value proposition, and primary user action.
  • Swap the target audience with your specific user persona including their technical sophistication level.
  • Change the platform specification to match your tech stack (React, Shopify, WordPress, custom).
